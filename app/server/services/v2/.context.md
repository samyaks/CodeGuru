## purpose

The pure logic layer behind the v2 (Takeoff) Gaps and Shipped tabs.
Lives in four files plus this doc:

- `gap-mapper.js` — translate v1 `suggestions` rows into v2 Gap shape
  consumed by the API + UI. Owns the kebab/snake status translation.
- `gap-prompt-generator.js` — wrap Claude calls that produce a Cursor
  prompt for an accepted gap (or a refined one).
- `gap-matcher.js` — match an incoming commit to an open Gap using a
  4-tier strategy (commit-ref, file-overlap, keyword, [Claude]).
- `gap-verifier.js` — given a matched (gap, commit) pair, decide
  whether the commit fully / partially / can-not-tell resolves the
  gap. Fail-safe contract: any failure → `'pending'`.
- `shipped-runner.js` — glue between webhook payloads, the matcher,
  and the verifier. Writes `shipped_items` rows + updates the source
  suggestion's `v2_status` / `verification`.

## constraints

- All Anthropic calls go through `app/server/lib/anthropic-tracked.js`.
  Never construct a raw `Anthropic` client.
- The verifier MUST return `'pending'` (not `'partial'`) when:
  - Claude is unreachable
  - Claude returns malformed JSON
  - Claude returns a string we don't recognize as `'verified'` /
    `'partial'`
  Anything else corrupts the user's understanding of "shipped."
  *(Currently NOT honored — see followup #2.)*
- The matcher MUST NOT auto-create a `shipped_items` row when match
  confidence is below `MIN_CONFIDENCE` (0.7). *(Currently NOT honored
  — see followup #1.)*
- The status enum stored in the DB is snake_case (`'in_progress'`).
  Outgoing API + UI uses kebab-case (`'in-progress'`). Translation
  lives in `gap-mapper.js#v2StatusFor`. Any new helper that reads or
  writes `v2_status` MUST go through the mapper or be tested for both
  casings.
- Don't make these files require a network or DB at import time.
  They're imported by tests and by routes that mock dependencies.

## decisions

### 4-tier matcher with explicit confidences
Rather than a single ML classifier, we use commit-ref (1.0),
file-overlap >50% (0.9), keyword + file-touch (0.8), and (planned)
Claude (≤0.7). Cheap tiers run first; we only burn an Anthropic call
when nothing else fires. The hard `MIN_CONFIDENCE = 0.7` lives in
`gap-matcher.js` so callers can't accidentally lower it.
agent · 2026-05-04

### Rule-based verifier pass before Claude
For `category === 'broken'` gaps with explicit `affectedFiles`, the
verifier first checks set-overlap. 100% overlap → `'verified'`,
partial → `'partial'` with the missing-files list. Only otherwise
does it call Claude. Saves cost on the most common gap type.
agent · 2026-05-04 · see `gap-verifier.js#ruleBasedBrokenCheck`

### `shipped-runner` runs sequentially per push
A 50-commit force-push shouldn't fan out 50 parallel Anthropic calls.
`processPush` awaits each commit before the next so we don't stampede
the verifier. Tradeoff is wall-clock latency for large pushes; we
accept that for cost predictability.
agent · 2026-05-04
