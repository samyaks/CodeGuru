## purpose

The v2 design-system components rendered by `pages/v2/Project.tsx` and
`pages/v2/StyleGuide.tsx`. Mirrors the prototype at
`docs/v2-reference/takeoff-prototype.jsx`. All components are
presentational â€” no data fetching, no router calls.

## constraints

- Tailwind classes only. No inline styles except for dynamic widths
  (e.g. `ProgressBar`'s `width: ${value}%`).
- Use `stone-*`, `red-*`, `amber-*`, `emerald-*` palettes from Tailwind
  defaults. The v2 design intentionally diverges from the legacy
  `--color-brand` (sky-900) tokens used by v1 pages.
- Lucide icons only. Match prototype icon choices exactly.
- Use `v2-font-serif` / `v2-font-sans` classes from `styles/v2-tokens.css`
  rather than hardcoding `font-family`.
- Status enum is kebab-case (`'in-progress'`, not `'in_progress'`). The
  backend translates at the API boundary in
  `app/server/services/v2/gap-mapper.js`.
- Components must accept `className` and forward extra props where
  reasonable so the parent page can adjust spacing without forking.

## decisions

### Single barrel export at `components/v2/index.ts`
Every component + its exported types re-export through `index.ts`.
Pages import from `'../../components/v2'`. Keeps the import surface
small and lets us rename internal files without churning consumers.

### `GapCard` owns refine-flow local state
`refining` and `refineText` live inside `GapCard`, not the parent. The
parent only sees the final `onRefine(id, instructions)` callback.
This keeps the gap list owner-agnostic â€” both `pages/v2/Project.tsx`
and the StyleGuide can render a gap without re-implementing the
refine textarea.

### `ChatDrawer` is fully controlled
Messages, open state, and quickPrompts come from props. No internal
message store. This makes it trivial to swap the stub in
`pages/v2/Project.tsx` for a real Claude-streaming implementation
later without touching the component.
