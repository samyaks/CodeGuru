## purpose

CodeGuru (rebranding to UpdateAI / Takeoff) is a web app that helps vibe
coders and PMs finish the last 40% of their AI-built apps. It analyzes
GitHub repos, surfaces broken / missing functionality / missing
infrastructure as a triaged "Gaps" list, generates Cursor prompts to fix
each gap, and verifies fixes by watching commits via GitHub webhook.

The active app lives entirely under `app/` and `packages/`. Everything
under `archive/` and the `archive/legacy-snapshot` git branch is retired
prototype work and is excluded from the analyzer scan.

## constraints

- Backend is CommonJS `.js` (Node + Express). Never `.ts` for backend
  services or routes.
- Frontend is Vite + React + TypeScript + Tailwind v4. Lives at
  `app/client/src/...`.
- Anthropic calls go through `app/server/lib/anthropic-tracked.js` so
  cost + tokens are recorded per analysis.
- The analyzer must read real files via the GitHub API. Never clone
  repos. Never invent analysis.
- The `analyze` endpoint must work without login.
- v1 and v2 APIs coexist until Phase 6b cutover; do not break
  `/api/suggestions` or `/api/projects/:id` while v2 ships.

## decisions

### Strangler-fig migration to v2 (Takeoff)
Build v2 alongside v1 with feature-flag gating, not a rewrite. The v2
project page lives at `/v2/projects/:id`. After Phase 6a, `/projects/:id`
defaults to v2 with `?v1=true` as the escape hatch.
@samyak (decision) ¬∑ agent (executed) ¬∑ 2026-05-04

### Reuse v1 `suggestions` table; add v2_-prefixed columns
Adding parallel columns (`v2_category`, `v2_status`, `verification`,
`v2_refined_from_id`, `v2_rejected_reason`, `v2_committed_at`) was
chosen over a new `gaps` table so v1 `/api/suggestions` keeps working
during the migration. Cutover happens in Phase 6b.
agent ¬∑ 2026-05-04 ¬∑ see `app/server/migrations/010_v2_gap_fields.sql`

### Match v2 status enum kebab-case at the API boundary
DB stores `in_progress` (snake) for SQL convention. API and frontend
use `in-progress` (kebab) because that's what the prototype's
`GapCard` GapStatus type expects. Translation lives in
`app/server/services/v2/gap-mapper.js#v2StatusFor`.
agent ¬∑ 2026-05-04

### Rename `archive` from a directory pattern to a SKIP_DIRS entry
Phase 0 moved `code-visualizer-mvp/` into `archive/`. The analyzer's
`SKIP_DIRS` set now includes `archive` so retired-but-tracked code
doesn't inflate module counts or readiness scores. Audit-self confirms
0 archive paths leak into the live tree.
agent ¬∑ 2026-05-04 ¬∑ see `app/server/services/analyzer.js`

### Defer the branding decision (Takeoff vs CodeGuru vs UpdateAI)
The v2 prototype uses "Takeoff" in user-facing copy. The repo is named
CodeGuru and `CONTEXT.md` flags an UpdateAI rebrand. All v2 components
preserve prototype copy verbatim. Branding will be picked after Phase 5
once the new IA is live in production for ‚â•1 week.
@samyak ¬∑ 2026-05-04

### Webhook handler extends, doesn't rebuild
v2's commit-matcher and verifier hook into the existing
`app/server/routes/github-webhook.js` via `setImmediate(processPushV2)`.
The v1 commit-review pipeline keeps running unchanged. Both subscribe
to the same signature-verified payload.
agent ¬∑ 2026-05-04
